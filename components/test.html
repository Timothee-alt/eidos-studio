Identité & direction artistique Concept fil rouge : une idée unique (symbole, mot, comportement, métaphore) répétée sur tout le site, pas seulement par section. Typo : remplacer ou compléter le duo « Space Grotesk + mono » par au moins une display ou une texte plus caractérielle (licence, fichiers, fallbacks). Palette : sortir du bleu « studio par défaut » ou le cantoner (accent rare, dominantes plus singulières). Photographie / 3D / illustration : si le site est 100 % UI + WebGL générique, ajouter une couche d’assets qui racontent Eidos (même minimaliste). Micro-détails : états hover/focus/active cohérents partout, transitions qui ne se répètent pas mécaniquement d’une section à l’autre. Narration & contenu Storyline : arc clair en 30 secondes (« qui / pour qui / preuve / next step ») sans dépendre du scroll infini. Preuves : études de cas avec contexte, contrainte, solution, résultat (même courts) — pas seulement visuels + chiffres décoratifs. Ton : une voix reconnaissable (humour sec, précision technique, poétique bretonne, etc.) — pas le blabla agence interchangeable. SEO inchangé mais humain : titres Hn et textes alternatifs utiles, pas que des keywords. UX & parcours Réduire la fatigue motion : arbitrer entre preloader + Lenis + hijack + horizontal + 3 scènes WebGL — garder ce qui porte le concept, couper le reste. Signaux de scroll : quand une section « prend » le scroll, indiquer clairement sortie / durée / interaction (copy, UI, progress). Mobile first réel : pas seulement responsive — priorités (quoi montrer en premier, quoi désactiver ou simplifier). Navigation : accès rapide aux projets et contact sans tout refaire passer par le menu cinéma si ce n’est pas volontairement conceptuel. Motion & interaction Courbes & rythme : une famille d’easing et de durées documentée (évite l’effet « démo GSAP »). Reduced motion : couvrir tout (WebGL simplifié ou statique, pas seulement CSS/GSAP partiels). Pas d’effet gratuit : chaque animation répond à « ça dit quoi sur la marque ? » Performance & fiabilité Budget : objectifs explicites (LCP, INP, TBT sur mobile milieu de gamme) et mesure sur build prod. WebGL : lazy load / désactivation hors viewport ; une scène partagée si possible ; résolution adaptative ; dpr limité. Assets : images WebP/AVIF, tailles explicites, pas de dépendances externes non critiques (ex. bruit distant) — inline ou hébergé. Fonts : sous-ensembles, display: swap, éviter le flash ou le blocage. Accessibilité & confort Curseur custom : option sans cursor: none global, ou désactivation si préférences / clavier / tactile. Focus visible partout (composants custom, menu overlay, liens géants). Contrastes : petits textes mono sur fonds sombres ; vérifier WCAG sur les états « muted ». Clavier : piège de focus dans le menu plein écran, Escape, ordre de tab logique. Lecteurs d’écran : titres de section, aria sur carrousels/strips, annonces si besoin pour changements d’état. Qualité front & craft Cohérence : tokens (couleurs, rayons, espacements) — moins de « one-off » en inline styles sauf exception. États : loading / erreur / empty pour contenus dynamiques si applicable. 404 / légal : page 404 au niveau du reste du site ; mentions / cookies si EU — les jurys regardent le détail. i18n : si un jour EN, structure prête (même minimal).
