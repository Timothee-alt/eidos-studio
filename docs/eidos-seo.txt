import type { NextConfig } from 'next'

const nextConfig: NextConfig = {

  compress: true,

      {
        source: '/(.*)',

          { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },

        source: '/_next/static/(.*)',

      },
    ]
  },
}

Checklist Core Web Vitals


LCP (Largest Contentful Paint) < 2.5s
? Fonts avec display: swap · Images hero avec priority={true} · Pas de JS bloquant


CLS (Cumulative Layout Shift) = 0
? Dimensions explicites sur toutes les images · Pas de font swap visible · Skeleton loaders


INP (Interaction to Next Paint) < 200ms
? Animations CSS only (scan-line, pulse) · Framer Motion lazy · Pas d'event listeners lourds


Fonts — éviter le FOUT
? Utiliser next/font/google · preload automatique · display: swap dans tous les cas

## SEO — Rčgles strictes

### Metadata
- Chaque page doit exporter une const metadata: Metadata
- title toujours au format : 'Page · Eidos Studio'
- description entre 140 et 160 caractčres exactement
- alternates.canonical obligatoire sur chaque page
- openGraph.images toujours défini (fallback /og-image.png)

### Structured Data
- Le composant <StructuredData /> doit ętre dans le <head> du layout.tsx
- Ne jamais dupliquer les @id dans les schemas
- Remplacer les PLACEHOLDER avant le déploiement (tel, email, geo)

### Performance
- Toutes les images : utiliser next/image avec width, height et alt explicites
- Images hero : ajouter priority={true}
- Fonts : uniquement via next/font/google, jamais de <link> manuel
- Pas d'import CSS externe (tout dans globals.css ou Tailwind)

### Contenu
- H1 unique par page, contient le mot-clé principal
- H2 contiennent des variantes de mots-clés secondaires
- Alt text sur toutes les images : descriptif + contexte
- Liens internes entre les pages de services

### Fichiers obligatoires avant déploiement
- /public/robots.txt
- /app/sitemap.ts
- /public/site.webmanifest
- /public/favicon.ico + icon-16.png + icon-32.png + apple-touch-icon.png
- /app/opengraph-image.tsx (OG image dynamique)
